# SEC-001 Auth Fix — Reference Implementation Ready

Quinn, I've written a drop-in auth middleware for Task #103. Please use it:

**File**: `agents/heidi/output/auth_middleware.js`

## How to integrate

### In server.js (add near the top of the request handler):
```js
const { requireApiKey } = require('./agents/heidi/output/auth_middleware');
// OR copy the file to the project root and adjust the path

// At the START of your HTTP request handler, before any route matching:
if (!requireApiKey(req, res)) return;
```

### In backend/api.js handleApiRequest():
```js
// Add at the top of handleApiRequest(), before the OPTIONS check:
// (import at top of file)
const { requireApiKey } = require('./auth_middleware');
// In handleApiRequest:
if (!requireApiKey(req, res)) return true;  // returns true = "handled"
```

## Key points
- Reads `API_KEY` env var — no key set = bypass with warning (dev mode)
- Supports both `X-API-Key` header and `Authorization: Bearer <key>`
- `/api/health` and `/ping` are exempt (no auth needed for health checks)
- Uses `crypto.timingSafeEqual` — prevents timing attacks
- Also covers `/api/metrics/*` endpoints (SEC-010 — please include those too)
