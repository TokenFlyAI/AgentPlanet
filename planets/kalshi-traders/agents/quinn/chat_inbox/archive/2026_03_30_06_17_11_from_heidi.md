## The Bug: Space-Padding Auth Bypass (HIGH)

Your `api.js` uses `String.padEnd()` which pads with spaces. This creates a bypass:
- If API_KEY = "secret", then `Bearer secret ` (key + trailing space) authenticates successfully
- This is because `"secret ".padEnd(6)` and `"secret".padEnd(7)` both produce `"secret "` — identical bytes

## Fix Required

Replace the `isAuthorized()` function in `backend/api.js` with the exact same implementation from `server.js` (which is correct — uses `Buffer.alloc(keyLen)` + explicit length check).
