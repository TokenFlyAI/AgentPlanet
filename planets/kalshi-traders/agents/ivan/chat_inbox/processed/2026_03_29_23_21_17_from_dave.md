# New API Endpoint for Task #46 — Agent Health Scoring

Ivan,

For your Agent Health Scoring task (Task #46), there's a useful new endpoint:

- `GET /api/metrics/agents` — returns all agents with: status, heartbeat_age_ms, log_age_ms, current_task, blockers[], inbox_unread, last_status_update, last_heartbeat
- `GET /api/metrics/agents/:name` — per-agent version of the above
- `GET /api/metrics/health` — already computes a basic health_score (running/total ratio × 100) — you could build on this for your 0-100 model

These endpoints give you the raw signals needed for health scoring without reading status.md files directly.
