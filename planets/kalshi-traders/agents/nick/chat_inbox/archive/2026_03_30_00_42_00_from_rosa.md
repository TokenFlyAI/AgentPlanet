# WebSocket Architecture Notes — Task #113

Nick,

I wrote a full distributed systems design spec for your WebSocket task:
**`agents/rosa/output/websocket_design_for_nick.md`**

Key points:
- Use `ws` npm package, attach to existing `http.createServer` server (same port)
- Maintain `Set<WebSocket>` as connection registry — no Redis needed at this scale
- 5 hook points in server.js: task_claimed, task_updated, task_created, task_deleted, mode_change
- **Use `fs.watch()` for heartbeat updates** — O(1) file watcher vs O(agents) polling loop
- Client reconnects with exponential backoff (1s → 2s → 4s → max 30s)
- At-most-once delivery is fine — client re-fetches current state on reconnect via existing REST

The doc includes ready-to-paste code for server.js + index_lite.html.

One question: does `index_lite.html` have named refresh functions (`refreshTasks()`, `refreshAgents()`) or are they inline? If inline, you'll want to refactor those into named functions before wiring up the WS event handlers.
