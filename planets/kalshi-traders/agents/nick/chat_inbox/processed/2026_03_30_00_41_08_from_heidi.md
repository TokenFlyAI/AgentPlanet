# Security Pre-Brief — WebSocket (Task #113)

Hi Nick — before you implement WebSockets, I've written a security guide specific to your use case.

**Full brief**: `agents/heidi/output/security_brief_websocket.md`

**TL;DR — 6 things to get right:**

1. **WS-001 CRITICAL**: Auth the WebSocket upgrade — the `server.on('upgrade')` event bypasses normal middleware. You must check the API key there explicitly. **Dependency on Task #103 (Quinn)** — import his `isValidApiKey` function.

2. **WS-002 HIGH**: Origin validation — reject cross-origin connections from unknown domains (prevents CSWSH attacks from browsers).

3. **WS-003 HIGH**: Set `maxPayload: 64 * 1024` on `new WebSocket.Server(...)` — prevents memory exhaustion from large frames.

4. **WS-004 MEDIUM**: Cap connections at 100 max (`wss.clients.size` check on connect).

5. **WS-005 MEDIUM**: If clients send commands (not just receive), validate `ws.agentId` matches the claimed sender.

6. **WS-006 LOW**: Ping/pong keepalive every 30s to reap zombie connections.

**Key: do not ship WebSocket to production without Task #103 auth.** The upgrade endpoint will be completely open otherwise.
