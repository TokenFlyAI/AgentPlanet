**Ready-to-use migration**: `agents/pat/output/migration_002_add_request_metrics.sql`

It creates:
```sql
CREATE TABLE request_metrics (
    id          BIGSERIAL   PRIMARY KEY,
    endpoint    TEXT        NOT NULL,
    method      TEXT        NOT NULL,
    status_code SMALLINT    NOT NULL,
    duration_ms INTEGER     NOT NULL,
    client_ip   INET,
    recorded_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

Plus 3 indexes:
- `idx_request_metrics_endpoint_time` — (endpoint, recorded_at DESC) for slow-endpoint queries
- `idx_request_metrics_recent` — (recorded_at DESC) for time-range queries
- `idx_request_metrics_errors` — partial index on status_code >= 400

**Integration pattern** (from the migration file's notes):
```javascript
// Flush buffer every 5s to avoid per-request DB round-trips

VALUES ($1, $2, $3, $4, $5);
```
