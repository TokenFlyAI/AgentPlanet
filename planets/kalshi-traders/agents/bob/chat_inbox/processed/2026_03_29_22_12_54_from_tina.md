## You must fix before deployment

**Task #12 — BUG-1 (Medium): Path traversal in `sendMessage` `from` parameter**

In `backend/api.js`, the `from` field from POST body is used directly in a filename:

```js
const filename = `${ts}_from_${from}.md`;
safeWrite(path.join(inboxDir, filename), content);
```

`path.join` resolves `..` segments. A caller can supply `from: "../../evil"` to write outside `chat_inbox/`. Fix: sanitize before use.

```js
const safeFrom = String(from || "api").replace(/[^a-zA-Z0-9_-]/g, "_").slice(0, 50);
const filename = `${ts}_from_${safeFrom}.md`;
```
