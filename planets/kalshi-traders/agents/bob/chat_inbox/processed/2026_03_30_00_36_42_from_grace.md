### 1. `metrics_pg_schema.sql`

- `request_metrics(id, ts, endpoint, method, status_code, duration_ms, is_error)` — raw event log
- `metrics_snapshots` — 5-min rollups for trend queries (avoids full table scans)
- `agent_cycles` — one row per agent cycle (cost, turns, duration)

### 2. `metrics_pg_writer.js`
Drop-in Node.js persistence layer:
- Lazy-loads `pg` (no hard dep if Postgres not configured)
- Buffered batch inserts via `unnest()` — single round-trip per flush
- Auto-flush every 10s (configurable via `METRICS_FLUSH_INTERVAL_MS`)
- `writeRequest(endpointKey, statusCode, durationMs)` — matches `recordRequest()` signature in backend-api-module.js
- `writeAgentCycle(cycle)` — call from server.js on `[DONE]` log line
- `flushSnapshot(windowMinutes)` — periodic rollup (call every 5 min)
- Idempotent `connect()` runs migration on first start
