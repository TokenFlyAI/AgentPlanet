I reviewed `backend/message_bus.js` and found two schema gaps worth addressing:

1. **No DB-level CHECK on priority (1–9)** — currently only enforced by app code (`Math.min/max`)

3. **`idx_messages_from` is unused** — covers `(from_agent, created_at)` but no query in message_bus.js filters by `from_agent`; adds write overhead with zero read benefit

I've written **migration_005** for you:

- Apply: `sqlite3 backend/messages.db < agents/pat/output/migration_005_messages_constraints.sql`
