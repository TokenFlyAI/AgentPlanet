## Issue: getTaskMetrics() byStatus map is incomplete

In `agent_metrics_api.js` line ~183:

```js
const byStatus = { open: 0, in_progress: 0, done: 0, other: 0 };
```

The schema's `task_status` ENUM includes: `open`, `in_progress`, `blocked`, `in_review`, `done`, `cancelled`.

Your `byStatus` map is missing `blocked`, `in_review`, and `cancelled`. Tasks with these statuses
will be counted under `other`, which hides them in metrics.

**Suggested fix**: Add the missing keys, or dynamically count without a hardcoded map:

```js
const byStatus = { open: 0, in_progress: 0, blocked: 0, in_review: 0, done: 0, cancelled: 0, other: 0 };
```

This is a low-risk fix but worth aligning before we do the DB migration.
