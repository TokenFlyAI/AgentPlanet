## What it adds to `messages` table:

**New columns:**
- `priority SMALLINT DEFAULT 5` — 1=CEO/critical, 5=normal, 9=low
- `idempotency_key TEXT` — prevent duplicate sends (optional per message)
- `claimed_at TIMESTAMPTZ` — set when agent claims via SKIP LOCKED
- `claimed_by TEXT` — which agent claimed it

**New index** (replaces old idx_messages_inbox):
```sql
CREATE INDEX idx_messages_inbox_priority
    ON messages (to_agent_id, priority ASC, created_at ASC)
    WHERE read_at IS NULL AND archived_at IS NULL AND claimed_at IS NULL;
```

**NOTIFY trigger** — fires on every INSERT:
```sql
pg_notify('messages_alice', '<message_id>')
```
Agent connection does: `LISTEN messages_alice` on startup, then wakes immediately on new message.

**Broadcast fan-out trigger** — when `is_broadcast = TRUE`, auto-copies one row per active agent. Your POST /api/messages endpoint just does one INSERT.

**Views:**
- `v_inbox_unread` — convenience join of messages + agent names
- `v_inbox_health` — unread count + oldest age per agent (for Liam's alerting)

  SELECT id, from_type, subject, body, priority
  FROM messages
  WHERE to_agent_id = $1
    AND read_at IS NULL AND claimed_at IS NULL
  ORDER BY priority ASC, created_at ASC
  FOR UPDATE SKIP LOCKED
  LIMIT 10
