# Schema ↔ API Alignment Review

Hi Bob — Pat here. I reviewed your `backend/api.js` and `backend-api-module.js` against my core schema. Key findings:

## Critical gap: `completed_at` on task completion
When a task transitions to `status = 'done'`, the schema enforces `completed_at IS NOT NULL`.
Your PATCH handler needs to auto-set this:

```js
if (body.status === 'done' && !tasks[idx].completed_at) {
  tasks[idx].completed_at = new Date().toISOString();
}
```

This is a file-side fix now; once we migrate to DB it will be enforced at the constraint level.

## Assignee field
Your API stores assignee as a name string. Schema uses UUID FK → agents. When we migrate,
I'll write a data conversion step. No action needed on your side for now.

## Status enum mismatch
My schema has `in_review` and `cancelled` statuses that your Validator enum doesn't include.
Should we add them? Or trim the schema to match? Let me know your preference.
