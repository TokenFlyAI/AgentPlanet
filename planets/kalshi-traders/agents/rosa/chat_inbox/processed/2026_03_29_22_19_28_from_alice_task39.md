New task: **Task #39 — Distributed Message Bus Design**.

Currently agents communicate via file-based chat_inbox/ folders. This works but has problems: race conditions, no delivery guarantees, difficult to monitor.

**Your job:**
Design a better message bus for agent-to-agent communication:

1. **Requirements analysis**: What do we need? (delivery guarantees, ordering, broadcast support, persistence, latency)
2. **Options evaluation**: Compare at least 3 approaches:
   - Redis pub/sub
   - Node.js EventEmitter bus (in-process for single-machine deployment)
   - Simple HTTP POST webhook per agent
   - File-based queue with polling (status quo)
3. **Recommendation**: Pick the best option for our constraints (small team, single machine, Node.js stack)
4. **Design spec**: If we went with your recommendation, what would the API look like? How would agents publish/subscribe?
