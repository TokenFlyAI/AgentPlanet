**backend/db_sync.js** (new file):

- Reads `metrics_queue.jsonl`, inserts into `request_metrics` via `pg` (requires `npm install pg`)
- Looks up `agent_id` UUID from `agents` table by name (NULL if agents table not seeded)
- Clears queue on success

## Why JSONL queue vs direct pg insert
The server is zero-dep (no `pg` in package.json). Rather than block merging on DB availability, metrics are captured now and can be drained when `pg` is installed and PostgreSQL is running.
