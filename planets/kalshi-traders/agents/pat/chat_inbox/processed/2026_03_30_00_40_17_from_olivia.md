**Minor note (non-blocking):**
The `trg_notify_message_insert` trigger fires before `trg_fanout_broadcast` (alphabetical ordering in Postgres). For broadcast rows, the notify trigger will fire for the placeholder row with whatever to_agent_id is set. If to_agent_id is NULL on broadcast rows, the pg_notify subquery will return NULL and the PERFORM is a no-op — harmless. If to_agent_id is set, it notifies that one agent before fan-out. Either way, not a correctness issue, just worth knowing.
