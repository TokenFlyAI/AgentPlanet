Task 2 (Beta task) is done. While waiting for a new assignment I did security hardening on `server.js`. Three attack surfaces fixed:

## Fixes Applied

### 1. Body Size Limit (`parseBody`)
- **Problem:** No request body size limit — a large POST body buffers entirely in memory (OOM risk)
- **Fix:** 512 KB cap; rejects with HTTP 413 if exceeded; `.catch()` handler updated to distinguish 413 vs 500
- **Location:** `parseBody()` + server `.catch()` handler

### 2. Agent Name Path Traversal
- **Problem:** Agent names from URL like `../etc/passwd` were decoded and passed to `path.join(EMPLOYEES_DIR, name)` without validation — could read/write outside the agents directory
- **Fix:** `agentName()` helper validates `/^[a-zA-Z0-9_-]+$/`; all 9 agent route handlers (GET, POST) now return 400 on any name containing dots, slashes, or other unsafe chars
- **Location:** 9 routes: `/api/agents/:name`, `/api/agents/:name/log`, `/message`, `/stop`, `/ping`, `/start`, `/inbox`, sub-resources, `/lastcontext`

### 3. `from` Field Filename Injection
- **Problem:** `body.from` was used raw in filenames like `${timestamp}_from_${from}.md` — a value like `../../some/path` writes files outside the intended directory
- **Fix:** `sanitizeFrom()` strips all non-alphanumeric/underscore/hyphen chars and truncates to 64 chars; all 4 filename-writing endpoints patched (message, inbox, team-channel, broadcast)
- **Location:** POST endpoints for `/message`, `/inbox`, `/api/team-channel`, `/api/broadcast`

### 4. CEO Inbox Filename Validation
- **Problem:** `POST /api/ceo-inbox/:filename/read` decoded the filename from URL and used it in `fs.renameSync` without validation — path traversal could move files outside CEO inbox
- **Fix:** Rejects any filename not matching `^[\w-]+\.md$`

## Status
- `node --check server.js` passes
- Live server (port 3100) unaffected — fixes are backward-compatible for all valid inputs
- No new dependencies added
